# Notion Pomodoro Timer ⏱️

A minimalist, borderless, floating desktop Pomodoro Timer that integrates directly with **Notion Database Automations** using [ntfy.sh](https://ntfy.sh). It floats on top of your workspace, automatically starting, pausing, or stopping based on your task's status in Notion.

---

## Features

- 🛸 **Floating Borderless UI**: Stays topmost, keeping your current task in sight. Drag and place it anywhere.
- 🔗 **Notion Sync (via Webhooks)**: Automatically starts, pauses, resumes, or stops the timer when you update a task's status in Notion.
- 📝 **Inline Editing**: Click the task name or timer directly to manually type a task name or edit the remaining time.
- ⚙️ **Customizable Settings**: Set Pomodoro and break durations, change themes (light/dark/system), and map custom database status keywords (e.g., "In Progress", "Paused", "Done").
- 🔊 **Audio Feedback**: Play ticking sounds during work and distinct alarms when pomodoros or breaks finish.

## Setup & Usage

### 1. Installation
Clone the repository and install the dependencies:
```bash
pip install customtkinter requests
```

### 2. Run the App
Launch the app by running:
```bash
python main.py
```

### 3. Connect to Notion Database Automations
1. Open the **Settings** window by clicking the gear icon `⚙` on the floating widget.
2. Click **Copy URL** to copy your personal webhook URL (formatted as `https://ntfy.sh/notion_pomo_<unique_id>`).
3. Open your Notion database, click the **Lightning Bolt icon (Automations)**, and create a new automation:
   - **Trigger**: Status changes (e.g., Status set to "Ongoing" / "In Progress").
   - **Action**: Send Webhook.
   - **URL**: Paste the copied URL.
   - **Properties to include**: Select **Name** (task title) and **Status**. *Important: Only include these properties to keep the payload size small.*

## Configuration

Your configuration and secret channel ID are stored locally in your system's AppData directory:
`AppData\Roaming\NotionPomodoro\config.json`

This file is automatically generated on first run and is kept separate from the project files to protect your personal Notion webhook URL.
